# IMA-226 — Live for any operator

_slug: live-any-operator_

## Why

Ticket premise was "MIP-only live today; generalize." The eng review corrected the premise:
the live path (`_OperatorWorker` → `tileReady`/`pushReady` → plate + slider) is **already
operator-parametric** — it forwards `operator: str` as `projector=` to `write_plate`
(`_viewer.py:853`) and `project_plate` (`:866`), and `_engine._PROJECTORS` already holds a
second, fully implemented projector (`reference`, `_engine.py:59`) that is simply not surfaced
in the viewer's `_OPERATIONS` registry (`_viewer.py:410`).

What IS hardcoded is the **z-reducer output-shape contract** — one flat image per region,
Z collapsed to 1:

```
today's live path assumes:  operator(region) -> (T, C, 1, Y, X)   ONE image per region
  _on_well:820       well = image[0, :, 0]          <- t=0 AND z=0 hardcoded
  run_operator:1777  start_acquisition(ch, 1, ...)  <- nz=1 hardcoded
  run_operator:1778  [f"{r}:0" for r in _order]     <- one fov per region hardcoded
  _on_push:1843      register_array(0, idx, 0, ...) <- z index 0 hardcoded
  _check_disk:1809   est = wells x n_t x C x Y x X  <- assumes z collapsed
  write_plate path   _output.py:223                 <- RAISES unless shape[2] == 1

contract holds for:   z-reducer   (mip, reference)                    Nz -> 1
contract breaks for:  fov-reducer (stitch, IMA-222)                   Nfov -> 1
contract breaks for:  plane-op    (decon/bgsub/flatfield, 223-225)    Nz -> Nz
```

This ticket lands the **contract** (`consumes` on the projector registry, pulled forward from
IMA-210), the **proof** (a second live operator, `reference`, streaming end-to-end), and
**loud validation** (unsupported kinds raise, never render silently-wrong pixels). The full
plane-op and fov-reducer wiring lands with the tickets that ship those operators (IMA-223,
IMA-222) — the outside voice verified it cannot ship here: `_output.py:223` hard-rejects
`nz>1`, `select_fovs` (`projection.py:200`) has no "all FOVs" semantics, and stitch needs
per-FOV geometry a `project(planes)` seam cannot carry.

5. Push-failure visibility (D7): count `register_array` failures in `_on_push` (today

   (`core.py:1666`, cap 1024) evicts ~83% of a 1536wp×4ch run. There is **no cache-miss

   ndviewer_light gains an `on_array_miss` loader hook; SquidMIP registers a
   tensorstore-backed loader for the written `plate.ome.zarr` after `writtenReady`
   (reusing `_ComputedPlateWorker._read`). Preview runs (`save=False`) have no disk copy —
   documented limitation in the preview UI text, full fix out of scope.
7. Disk-estimate honesty (D10): `_check_disk` gains the operator-kind multiplier —
   fov-reducers estimated at zero-overlap (conservative upper bound, per the guard's own

   raise) but written+tested so IMA-222 inherits a correct guard.

## Acceptance / oracle

- A non-MIP operator (`reference`) streams partial results to the plate tiles AND the
  ndviewer slider as it computes — asserted through the stub's recorded `register_array`
  calls, not by eyeball.
- MIP on a real acquisition is byte-identical to pre-change output.
- A plane-op or fov-reducer key fails loud with the ticket name, never silently wrong.
- After a full-plate run, scrubbing back to the first well shows its planes (disk-backed
  miss loader), not a blank frame.

## NOT in scope (explicitly deferred)

- **Plane-op (nz=Nz) end-to-end wiring** → IMA-223: `_output.py:223` `_validate_image`
  hard-rejects `Z>1`; multiscales are YX-only; the reopen path (`_ComputedPlateWorker._read`
  `arr[0,:,0]`, `_open_computed` `n_z: 1` + hardcoded `np.uint16`, `_viewer.py:975/1683/1717`)
  is z-collapsed. Landing it without a real plane-op to validate against is design-on-faith.
- **Fov-reducer end-to-end wiring** → IMA-222/210: needs `select_fovs` "all" semantics for
  ragged FOV counts (`projection.py:200` raises), group-by-region engine loop, a geometry-
  carrying seam (stage positions + overlap — see `fov-axis-needs-geometry` learning), and a
  reworked memory contract (Nfov frames in flight per worker breaks the "~139 MB × workers"
  invariant and `test_engine.py:206`).
- **Stitch operator itself** → IMA-210's other half (tilefusion integration).
- **t-axis streaming** — `_on_push`/`_on_well` stream t=0 only; no Nt>1 dataset exists
  (existing TODO).
- **Preview-mode eviction** — a `save=False` run has no disk copy for the miss loader;
  documented limitation.
- **Distribution** — no new artifact type; existing packaging unchanged.
