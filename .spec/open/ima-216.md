# IMA-216 вҖ” Viewport -> tiles core

_slug: viewport-tiler_

## Why
The one real algorithm we write. Framework-neutral so web reuses it. Today PlateOverview
blits ONE full-plate QImage scaled to zoom (`_viewer.py` paintEvent) вҖ” no culling, no LOD;
this replaces that path's data side (render integration is IMA-218).

## Scope
`_tiling.py`, numpy only, Qt-free (precedent: `_layers.py`). Two separated pieces:

1. **`select_tiles(bbox_um, um_per_px, geometry) -> list[TileDescriptor]`** вҖ” PURE.
   LOD pick + frustum cull. Returns the **ideal** set (what should be on screen).
2. **`TileCache`** вҖ” stateful. Byte-budget LRU + keep-parent-until-child-ready.
   `resolve(ideal) -> renderable` substitutes cached ancestors for missing tiles.

### Contract (owned HERE; IMA-217 implements, IMA-218 consumes)
- **World space: stage micrometres.** bbox = `(x0, y0, x1, y1)` Вөm; zoom = Вөm-per-screen-pixel.
  Callers convert at the edge (viewer: from `_cd`/`_ox`/`_oy`; matches `_multiscales` Вөm scales).
- **`TileDescriptor`** вҖ” frozen dataclass: `(level: int, key: Hashable, channel: str, bbox_um: tuple)`.
  Tiles are cached **pre-composite per channel** (contrast/LUT changes invalidate nothing;
  compositing is the renderer's job).
- **`Geometry`** вҖ” per-level: `scale_um_per_px: float`, `bboxes: (N,4) float64 contiguous Вөm array`,
  `keys: sequence`. Levels need NOT align power-of-two across the ladder вҖ” plate-level tiles
  (coarse) and per-FOV pyramid levels (fine) coexist as independent levels; regridding at the
  seam is IMA-217's construction problem, invisible to the tiler.
- **`TileSource` Protocol** вҖ” `read_tile(desc) -> np.ndarray`. Synchronous. IMA-217 implements.

### Algorithm decisions (eng review 2026-07-20)
- **LOD pick:** smallest level with `scale_um_per_px <= requested um_per_px` (just finer than
  screen res); clamp both ends (finer than level 0 -> level 0; coarser than coarsest -> coarsest).
  Tie (`==`) picks that level exactly.
- **Coarse end:** a plate-level pyramid ABOVE the per-FOV pyramids (built by IMA-217) so
  fit-to-plate resolves to a handful of tiles вҖ” per-FOV pyramids bottom out at 256 px
  (`_output.py` `_PYRAMID_MIN_YX`), which can never satisfy O(viewport) at whole-plate zoom.
- **Cull:** ONE vectorized numpy overlap test on the level's `(N,4)` array -> index array.
  No per-item Python loop (acceptance criterion, not style).
- **Cache budget: bytes** via `arr.nbytes`, not tile count (level-0 FOV plane ~8 MB vs coarse
  tile ~128 KB). A single tile larger than the whole budget is admitted alone (never refused вҖ”
  refusing blanks the screen).
- **Parent retention:** a tile with an in-flight child is **pinned** (skipped by eviction);
  unpinned when the child lands or its fetch fails. Pins capped at half the byte budget;
  past the cap the oldest pending descriptor is dropped (degrades to blur, never deadlocks).
- **Fetch lifecycle is the CALLER's:** `mark_pending(desc)` / `insert(desc, arr)` /
  `fetch_failed(desc)`. No threads, no callbacks, no Qt inside `_tiling.py`.
- **Streaming acquisition:** `invalidate(predicate)` вҖ” a newly-written FOV invalidates its
  ancestor plate-level tiles; per-channel caching means contrast updates invalidate nothing.

## Acceptance / oracle
- `select_tiles` returns O(viewport) **ideal** descriptors (not O(all FOVs)) at every zoom
  including fit-to-plate (given a geometry with a plate-level coarse ladder).
- Picks the pyramid level just finer than screen res (ideal set; `resolve` may transiently
  substitute coarser cached ancestors вҖ” that is the feature, not a violation).
- Cull is a single vectorized numpy op; correctness asserted on a 55k-box synthetic geometry.
  Wall-clock timing lives ONLY under the `integration` pytest marker (repo precedent:
  `test_performance.py`), never as a hard CI assert.
- `TileCache` never exceeds its byte budget (mixed-size insert test) and never evicts a
  pinned parent (fill-to-capacity-with-pending-child test).

## Test plan (all pure-numpy, no fixtures needed)
- LOD: exact-tie, finer-than-level-0 clamp, coarser-than-coarsest clamp, single-level ladder.
- Cull: viewport inside / partial / fully outside plate (-> []), zero-area & inverted bbox
  (normalize or raise вҖ” pick raise: explicit > clever), empty geometry (0 FOVs), 55k-box
  correctness, descriptor determinism (stable ordering).
- Cache: LRU eviction order, re-access promotes, byte budget under mixed sizes,
  oversized-single-tile admit, capacity-0/1 degenerate, pin survives full cache,
  unpin on insert AND on fetch_failed (failed child must not leak a pin),
  pin-cap overflow drops oldest pending, invalidate(predicate) drops matching + their pins.
- resolve(): missing tile -> nearest cached ancestor; no ancestor -> absent (caller renders
  nothing for that slot); child landing removes parent from renderable set.

## Failure modes
- Ancestor evicted while child pending -> blank flash on pan. Guard: pinning; tested.
- Failed fetch leaks a pin -> cache slowly fills with immortals. Guard: unpin on
  fetch_failed; tested.
- Вөm/px mix-up at a call site -> subtly misaligned mosaic. Guard: single canonical space
  (Вөm) stated in the contract docstring; geometry validated (x1>x0, y1>y0, finite).
- NaN/inf stage coords from a bad coordinates.csv row -> NaN never compares true, tile
  silently never renders. Guard: `Geometry` constructor rejects non-finite bboxes (loud, early).
