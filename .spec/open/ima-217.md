# IMA-217 — OME-Zarr pyramid source

_slug: zarr-pyramid-source_

## Why
The B1 keystone (IMA-187) needs a tile source: the viewport tiler (IMA-216, `_tiling.py`,
`viewport(bbox, zoom) -> tiles`) picks a level and asks for pixel tiles. `_output.py`
already WRITES a per-FOV pyramid (levels 0..L, 2x block-mean, 1024² chunks) but the only
read path is ad-hoc and viewer-local (`_ComputedPlateWorker._read`, _viewer.py:971-975 —
raw tensorstore opens at two hardcoded levels, whole-plane reads, no cache, no pending
semantics). `PyramidSource` becomes the canonical read path; IMA-218 (which edits
`_viewer.py`) migrates the viewer onto it. Builds on the shipped OME-zarr writer
(IMA-184/185).

Full rationale, premises, alternatives: design doc
`~/.gstack/projects/maragall-SquidMIP/julioamaragall-juliomaragall-ima-217-zarr-pyramid-source-design-20260720-231952.md`
(APPROVED; 3 adversarial review rounds + outside-voice pass, 19+17 findings resolved).

## Scope
One new class `PyramidSource` in `_output.py` (ticket constraint: edit `_output.py` only),
plus a small write-path refactor to feed it. numpy in / numpy out, framework-neutral.
tensorstore only — no new deps.

## Contract (locked)

```python
class PyramidSource:
    """Tile-read API over a (possibly still-writing) plate.ome.zarr."""

    def __init__(self, plate_dir, *, cache_bytes: int = 512 << 20):
        # Lazy — plate_dir may not exist yet (live wiring constructs the source
        # BEFORE write_from_stream creates the directory). Nothing opened here.

    def wells(self) -> dict[str, list[int]]:
        # {region: [fov, ...]} from plate/well group metadata. This is the PLANNED
        # layout (groups are written up front, _output.py:309-319) — includes FOVs an
        # aborted run never wrote; levels() is per-FOV availability. {} until the
        # plate group zarr.json exists AND parses. Parse-tolerant SNAPSHOT: a well
        # whose group zarr.json is missing/unparseable (the plate json is written
        # before the per-well jsons) is OMITTED this call and appears once written.

    def levels(self, region: str, fov: int) -> list[tuple[int, int]]:
        # (Y, X) per written level, level 0 first. [] if the FOV isn't AVAILABLE.
        # Availability gate = field group zarr.json EXISTS AND PARSES (write_group is
        # plain write_text, so a truncated json observed mid-write means PENDING,
        # never a raise). _write_field writes arrays first, group json LAST, so a
        # parseable group json proves a complete field (in-process guarantee; NOT a
        # crash/power-loss guarantee — no fsync). Level COUNT from that json's
        # multiscales[0].datasets; per-level (Y, X) from each level array's own
        # {i}/zarr.json "shape". Cheap JSON parses, no tensorstore open.
        # MEMOIZATION RULE: availability + level count + shapes MAY be memoized per
        # (region, fov) after first success; the ONLY invalidation is ingest() for
        # that FOV (see below). External/cross-process rewrite of the plate while a
        # source is open is OUT OF CONTRACT.

    def read(self, region: str, fov: int, level: int,
             ys: slice, xs: slice, t: int = 0) -> "np.ndarray | None":
        # (C, h, w) native dtype = arr[t, :, 0, ys, xs] of the 5-D level. Slices are
        # level-local pixels, CLAMPED (fully out-of-range -> empty array, not None);
        # negative/stepped slices unsupported (undefined). Malformed region id ->
        # ValueError (from parse_well_id — the third documented outcome).
        # Read order:
        #   1. RAM hit -> slice-copy INSIDE the lock (coarse arrays are small; this
        #      closes the eviction/slice race) -> return;
        #   2. availability gate fails -> None (pending — tiler keeps parent; t is
        #      NOT validated on a pending FOV);
        #   3. level >= the gate's memoized level count -> None (level the field
        #      doesn't have — small field, or a shrunken rerun whose stale
        #      higher-level dirs must NEVER be opened; levels() disambiguates
        #      from pending);
        #   4. t bounds-checked against the level's T extent -> IndexError
        #      (normalized — tensorstore's own error type never escapes);
        #   5. tensorstore read. After gate + level bound, ANY open/read failure
        #      raises — there is no legitimate absence left (no error-text
        #      matching, no except-all).
        # HONEST LIMIT (zarr semantics): absent chunk FILES read as fill-value
        # zeros, silently — chunk loss is undetectable at this layer. "Corrupt
        # raises" means unparseable array metadata or decode failure of present
        # bytes.
        # Warm-cache rerun caveat: a RAM hit may serve STALE-BUT-COMPLETE prior-run
        # data mid-rewrite (never partial); ingest on rewrite completion replaces.
        # RAM is AUTHORITATIVE for ingested FOVs even if the current wells()
        # snapshot (e.g. after a regions-subset rerun rewrote the plate json)
        # no longer lists them.

    def ingest(self, region: str, fov: int, levels: "list[np.ndarray]") -> None:
        # Write-path hook: cache the coarse tail of an already-computed pyramid.
        # `levels` = the FULL _pyramid list of 5-D (T, C, 1, Y, X) arrays; list
        # position == 0-based disk level index. FIRST drops ALL prior state for
        # this FOV — cached levels, memoized availability/count/shapes, open
        # tensorstore handles — THEN inserts (this is the single invalidation
        # point that makes memoization + reruns + shrunken pyramids safe).
        # Thread-safe (writer threads call this; GUI thread calls read).

    def close(self) -> None:
        # Drop level cache + handles + memoized metadata (release RAM at run end).
        # Source stays usable: a later read lazily re-derives from disk.

    def read_tile(self, desc) -> "np.ndarray":
        # IMA-216 TileSource Protocol adapter (216's _tiling.py owns the interface;
        # this implements it for PER-FOV levels — see 216's spec, pushed 3eb5335).
        # desc.key == (region, fov); desc.level == per-FOV disk level index;
        # desc.channel is a channel LABEL resolved to a C-index via the field group
        # json's omero.channels order (== the C-axis order, _output.py module doc).
        # Returns the FULL level plane for that channel: 2-D (h, w) native dtype
        # (per-FOV tiles are one-tile-per-level in 216's geometry; desc.bbox_um is
        # the Geometry's concern, not a lookup key).
        # Raises LookupError on a pending/absent tile or unknown channel label —
        # 216's cache has no None state; the 218 fetch executor maps the raise to
        # TileCache.fetch_failed (keep-parent). Wraps read(); same gate semantics.
```

Key normalization: cache/handle keys use the `parse_well_id`-canonical region
(uppercased letters+digits) inside `ingest`/`read`/`levels` — "b2" ingest and "B2"
read hit the same entry.

### Cache policy (numbers locked)
- Cache exactly levels with `index > 0 AND max(Y, X) <= 1024` (one chunk). For the
  real 4168-px ladder (4168/2084/1042/521/260/130) that caches 521+260+130 ≈ 2.8 MB/FOV
  — the few-well zoom band the mosaic actually pans in, not just thumbnails. Level 0
  NEVER cached, even when itself ≤1024 (tiny/test fields cache nothing; disk serves
  them).
- Coarsest CACHED level per FOV is pinned (130-px coarsest ≈ 135 KB at 4ch uint16;
  1536 FOVs ≈ 208 MB, 1536×4 FOVs ≈ 830 MB worst case — the whole-plate mosaic paints
  from pins).
- Pinned bytes COUNT toward `cache_bytes` (default 512 MiB). Pinning is best-effort:
  if pins alone exceed budget, demote pins in LRU order (least-recently-READ pinned
  entry first) — degrades to disk, never OOM, correctness unaffected. Accounting =
  `arr.nbytes` of cached 5-D arrays (T=1 in practice).
- SIZING RULE for the live wiring (218): pass `cache_bytes >= n_fields ×
  coarsest_level_nbytes × 2` — computable up front from metadata — so a 4-FOV 1536
  plate keeps all pins. The default suits ≤1536 fields at 1 FOV/well.
- Non-pinned levels LRU-evict to `cache_bytes`.
- tensorstore handles: small LRU (~128) keyed (region, fov, level); dropped for a FOV
  on its ingest() and on close().
- ONE `threading.Lock` guards level cache + byte count + handle LRU + memoized
  metadata; tensorstore open/read happen OUTSIDE it (benign double-open discards the
  loser); RAM slice-copy happens INSIDE it (see read order).

### Write-path wiring
- Refactor `_write_field` to RETURN the levels list (today returns `len(levels)`,
  _output.py:253); `_write_one` calls `source.ingest(region, fov, levels)` right after
  the write (no pyramid recompute) and reports `len(levels)` upward — the manifest's
  `levels` count is unchanged behavior (regression-guard it).
- `write_from_stream(..., source: PyramidSource | None = None)`; `write_plate` gains
  the same param and passes it through (the live GUI path enters via `write_plate`,
  _viewer.py:853). `on_well` unchanged.
- Rerun invariant: `_write_field` UNLINKS the field group zarr.json BEFORE writing
  arrays — a prior run's json must not make a mid-rewrite field look available.

### Data flow
```
LIVE RUN                                        COLD REOPEN
project_plate ─► write_from_stream              PyramidSource(plate_dir)
     │   _write_one:                                  │ wells(): plate/well group json
     │     levels = _pyramid(image)                   │ levels(): field json gate + shapes
     ├──► disk: field/{0..L}  (unchanged)             ▼
     └──► source.ingest(FULL list) ──► RAM       read(region,fov,level,ys,xs)
              (drop-then-insert:          │           │ RAM hit ──► slice (in lock) ─► (C,h,w)
               the invalidation point)    │           ├ gate fail ──► None (keep-parent)
                                          │           ├ level ≥ count ──► None
        tiler (IMA-216) ◄─────────────────┴───────────┴ ts read (post-gate errors RAISE)
        viewport(bbox,zoom) -> read(...)
```

## Build items (one per iteration, test-first, in order)
1. `PyramidSource` skeleton: lazy `__init__`, `wells()` (parse-tolerant snapshot),
   availability gate helper, `levels()` (gate + shape parse + memoization rule).
2. `read()` disk path: gate → level-count bound → t bound → handle LRU → clamped
   slice → (C,h,w); None/ValueError/IndexError/raise semantics; key normalization.
3. `ingest()` (drop-then-insert invalidation) + RAM-first `read()`: cache rule ≤1024,
   pin, byte budget, LRU + pin-demotion-in-LRU-order, `close()`.
4. Write-path wiring: `_write_field` returns levels + group-json unlink; `_write_one`
   ingest; `write_from_stream` AND `write_plate` `source` param; manifest regression
   guard.
5. `read_tile(desc)` TileSource adapter: key/level/channel-label mapping, LookupError
   semantics (duck-typed desc in tests — do NOT import `_tiling.py`; it lives on the
   unmerged IMA-216 branch).
6. Integration: live-run read-during-write test (below) + thread-safety smoke.

## Tests (new file `tests/test_pyramid_source.py`; reuse test_output.py's fabricated
metadata + stream helpers)
- Pixel-exact reads at every level vs written arrays; slice window == numpy slice of
  the level array.
- RAM-hit proven: read served after deleting the disk level arrays (a 600-px field
  yields levels 600/300/150 — 300 AND 150 cached, pin = 150; also assert the 1024-px
  band boundary: a 2100-px field's 1050-px level 1 is NOT cached, its 525-px level 2 is).
- None-for-pending vs None-for-out-of-range level (levels() disambiguates); arrays
  present but NO group json → pending; truncated/empty field group json → pending, no
  raise.
- wells(): PLANNED layout incl. never-written FOVs; {} on missing plate_dir (lazy
  constructor); plate json present but a well group json absent → that well omitted,
  no raise (snapshot stabilizes once written).
- Budget eviction incl. pin-demotion (LRU order among pins, deterministic) under tiny
  cache_bytes; re-ingest replaces without double-count.
- Region-key normalization ("b2" ingest / "B2" read hits RAM); malformed region id →
  ValueError.
- Rerun: COLD source — group json unlinked first → pending mid-rewrite, complete
  after; WARM source — stale-but-complete served, then ingest drops old cache +
  handles + memoized count and the new (SMALLER, fewer-levels) pyramid is what reads —
  the stale higher-level dirs on disk are never opened (level ≥ new count → None).
- Corrupt store: garbage bytes in a chunk file of a gate-passed, in-range level →
  read raises (decode failure). DELETED chunk file → fill-value zeros, silently
  (documented zarr semantics — test asserts zeros, not an error).
- Slice clamping incl. fully-out-of-range → empty array; out-of-range t on an
  AVAILABLE field → IndexError (normalized, not a tensorstore error type); any t on a
  PENDING field → None (gate first).
- close() drops cache/handles/memoized metadata; subsequent read lazily re-derives.
- Tiny field (≤256 px): single level, nothing cached, disk read works.
- Non-contiguous fov ids; multi-letter rows (AA3).
- read_tile: happy path (2-D plane == read()'s channel slice), pending FOV →
  LookupError, unknown channel label → LookupError, channel-label→index mapping
  follows omero order (duck-typed desc namedtuple; no `_tiling.py` import).
- REGRESSION: existing test_output.py suite green; manifest `levels` count unchanged
  with and without a `source`; `write_plate` accepts and forwards `source`.
- Integration (the tiler's live flow): `write_from_stream` with a `source` over a
  fabricated stream; concurrent reader thread polls `read()` during the stream —
  every result is None or pixel-complete, never partial; after drain, all FOVs
  available. Thread-safety smoke: concurrent ingest+read, no exception/deadlock.
